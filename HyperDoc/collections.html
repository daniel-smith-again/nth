Collections

A collection is a form of composite data which appears as a single unit, but is also deconstructible into components.

A collection is comprised of fields which are ordered according to numerical index values.

A collection contains a context such that a field may be bound by an identifier.

Constructing Collections

The Collection Constructor is a default expression which will construct a collection by resolving fields in the expression as fields in the collection.

The sequence of program components following the keyword collection are either a symbol, expression, or a sub-sequence of the form '(''Symbol . ''Program) which is referred to as a Key. If a symbol occurs in the sequence, it is resolved as an identifier and corresponds to a field containing the resolved value. If an expression occurs in the sequence, it is evaluated and the result is used as the value in a field. If a Key occurs in the sequence, it corresponds to a single field where the symbol preceding the . keyword is used as the identifier and the program following the keyword is evaluated and the result used as the value for the binding and for that field.

The Collection Constructor will perform any evaluation in a new scope which will contain any bindings created by Key sequences. Bindings which do not bind a field as a value will be discarded when the Collection Constructor expression is reduced.

Collection Expressions

Subscript

Subscript is a default expression which will locate a field with an index equal to a number.

Extract

Extract is a default expression which will locate a field bound with an identifier equal to a symbol.
