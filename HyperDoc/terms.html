Terms

Application

The operation which invokes a function (Refer to the section on Functions).

Binding

A function, from an identifier to a value, held as a field in a context

Binding

A function contained by a context which represents an association between a Program Model component and some data (Refer to the section on Bindings).

Blank

A character under the General Category Major Class Z in 4.5 General Category, Table 4-4 of the Unicode® Standard

Body

The expression which is evaluated by Inversion, Let or application to determine the result.

Character

An encoded numerical value according to the Unicode® Standard or the ASCII Standard corresponding to written text.

Closure

A context which contains at least every binding in the context which was used to execute the construction of the new context.

Collection

The construct provided by nth for creating structured data (Refer to the section on Collections).

Computer

The device or mechanism which is responsible for executing a program.

Context

The collection maintained by the environment which contains the bindings available for evaluation.

Continuation

The portion of an inversion expression body which is represented by the first parameter provided to the application of a modulator; The part of a program which is evaluated using the result of a function application.

Correct

The term describing a program which does not result in an error.

Data

A static information constant of some size occupying storage resources.

Default

Data or another resource which must be present as auxillary to the implementation (Refer to the section on Defaults).

Digit

A character under the General Category Major Class N in 4.5 General Category, Table 4-4 of the Unicode® Standard

Discard

An operation which will remove data from the environment such that it is inaccessible during execution and any resources claimed by the discarded data are unclaimed.

Environment

That which is provided by the computer to facilitate execution.

Error

A condition which invalidates a program and prevents execution.

Escape Sequence

A sequence of characters occurring inside a string syntax which corresponds to one or more characters to include in the string.

Execute

The process of carrying out effects in the environment according to a program.

Evaluate

An abstract process which determines effects of executing an program (Refer to the section on Evaluation).

Expression

A syntax which denotes an operational step (Refer to the section on Syntax).

Field

A signle component of a collection.

Function

The construct provided by nth for creating re-useable programs (Refer to the section on Functions).

Glyph

A character under one of the General Category Major Classes L, M, P, S, or C in 4.5 General Category, Table 4-4 of the Unicode® Standard

Handler

A function which is bound by the default function, with.

Identifier

The Program Model component which corresponds to a value according to a binding in the context for that program.

Image

The construct provided by nth for creating re-useable contexts (Refer to the section on Images).

Implementation

A program which acts according to the requirements given in this document (refer to the section on Conformance).

Member

A single component of a type.

Modulator

A function bound by inversion expression.

Number

A type of data representing a scalar.

Operation

A single executional step, denoted by an expression.

Parameter

A value used by application.

Program

An interpretable representation of the possible operations which may be executed, in some order, to accomplish a desired result.

Program

The instructions which execution uses to carry out effects in the environment.

Program Model

The data structure which forms the cannonical representation of an nth program.

Quote

A Quote is a Program Model represented as data which a program can operate on.

Reduce

An operation performed during evaluation which replaces a default expression with the data resulting from executing the corresponding operations for that expression.

Resolve

The operation which, given an identifier, searches the scope for a binding which associates a value with that identifier.

Resource

A facility of the environment that is consumed by the process of executing a program.

Rewrite

An operation performed during evaluation which applies a transform to an expression.

Scope

The context which is used by evaluation.

Semantics

The properties which constrain the evaluation of an expression corresponding to the operations required to execute a program.

Shadow

A binding in a scope may have an identifier which is equal to an identifier in a context which the scope closes over. In this case, the value of the binding in the scope will be used instead of the value of the binding in the other context.

Source

A program format representing a Program Model as a string.

Step

An expression which is evaluated as part of Sequential Evaluation.

Storage

The environment resource which is consumed when generating or holding information during execution of a program.

String

A collection of character values.

Subtype

A type for which the comparison (a < b) is successful, given that a is that type and b is a supertype.

Supertype

A type for which the comparison (a > b) is successful, given that a is that type and b is a subtype.

Symbol

An abstract, unique data value represented by a textual entity.

Syntax

A sequenced pattern of characters in a source which correlates to an individual Program Model component. (Refer to the section on Syntax).

Tail

The last step which is evaluated in a sequence or function body and consequently determines the result of the sequence or application.

Top Level

The image which is open while an expression is being evaluated.

Transform

A binding where the identifier is a Program Model component.

Type

The construct provided by nth for reasoning and constraining the possible instantiations of data in a program (Refer to the section on Types

Value

The data which replaces an identifier according to a binding in a context.
