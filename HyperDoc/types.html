Types

Types provide a means of specifying constraints on a program and to some extent, the intent of that program.

There are two forms of types provided by nth: "discrete" and "continuous".

A Discrete Type is a type with a finite amount of members such that every possible instance of a Discrete Type is a known member.

A Continuous Type is a type with an unknown amount of members such that for every known member, an instance of a new member of that type may still be created.

Correct Programs

A program is determined to be correct if there are no errors resulting from constraint violations.

A constraint is either conformed to or violated by a program according to a set of rules which specify how a constraint relates to a program.

Type Rules

  1. A value is an instance of some type which is instantiated in the scope.
  2. The result of evaluating an expression is a value
  3. Given a type which is a subtype of another, an instance of the first is an instance of the second.
  4. Given a continuous type defined with more than one "base" type, an instance of that type neccessitates at least one instance of each of the "base" types.
  5. Given a continuous type defined with multiple "base" types, instances of each "base" type must be instantiated for all to be an instance of that continuous type.
  6. An instance of a continuous type is an instance of each of its "base" types such that the application of the "proposition" function, using those instances as parameters, result in Success.
