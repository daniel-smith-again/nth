Images

An image is a representation of a context as data.

Images are used by evaluation such that a program may instantiate data or bindings to be used by further evaluation of other programs.

An image corresponds to a collection such that an image is constructed by constructing a collection.

The correspondence between a collection and an image is such that a field with a label corresponds to a binding in a context. A field with no label containing a function which is an instance of (Program -> *) corresponds to a transform.

Image Operations

All operations on images are bound to the identifier, !.

Load

The Load operation accepts a string parameter which represents the location of a stored image.

Load will locate the stored image, instantiate it, and produce it as a result.

If the location specified by the string is not an image, or there is no corresponding location, Load will result in Nil.

Store

The Store operation accepts a string and an image as parameters. It will save the image at the location corresponding to the string parameter.

If there is no corresponding location, the operation will have no effect.

Open

The Open operation accepts a single image parameter which it will set as the Top Level.

In the case that the Top Level has a binding to itself, executing Open using that binding will have no effect on the Top Level.

In the case that the image being Opened does not have a duration tied to the Top Level, its duration must be at least for the period where it is the Top Level.

Leave

The Leave operation accepts no parameters and will set the Top Level to the image which was the previous Top Level. If no previous Top Level exists, the session will exit.

Export

The Export operation minimally accepts a string parameter. It use the Top Level to create a stored object of some implementation defined format at the location specified by the parameter.

Export may accept additional implementation defined parameters for the purposes of supporting various object formats.

The Load operation must support all formats supported by Export such that Exported images may be instantiated in some capacity as if they were Stored images.

Transforms

A transform is a function which is applied to a program to rewrite it.

Given a program, any number of transforms may be applied in succession until a default expression results.

Describe

Describe is a default function which accepts either a single parameter of any type, or a string and another parameter of any type.

If given a single parameter, Describe will result in a string. If given two parameters, it will result in the parameter of any type.

Describe will maintain a list of key-value pairs stored in the Top Level where each pair is established by the application of Describe with two parameters. For each pair, the key corresponds to the first parameter and the value corresponds to the second.

When Describe is applied with one parameter, it will search the list for a pair with a key that is equivalent to the parameter. If one is found, then the string value for that pair is provided as the result. If the list does not contain a matching key, the list of each outer image is searched in succession until a key is found. If no matching keys exist in any lists, the result will be Nil.
