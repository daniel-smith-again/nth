Data

1-1

Data is the substance which operations are executed on.

1-2

A unit of data which has been instantiated will always consume resources in the form of a size measured in Bytes.

1-3

The resources consumed by an instantiated unit will be constant for the duration of that unit.

1-4

Discarding a unit must releast all resources which were consumed by instantiation.

Scalars

2-1

Scalars are types of data that have a close correspondence to resources in the environment.

2-2

Scalars may store whole, positive numerical values of a fixed range where the range of possible values depends on each Scalar type.

2-3

The minimum possible numerical value which a Scalar can store will be zero.

2-4

The default Scalar types are Byte and Word.

2-5

An implementation may provide additional Scalar types which must be distinct from all existing Scalar types and must comply with all properties of the default Scalar types.

2-6

A Scalar is instantiated with a default value which must be the minimum value for that type.

Bytes

3-1

The Byte is the smallest unit of data, such that one Byte corresponds to one resource in the environment.

3-2

The maximum possible value which can be stored in a Byte is implementation dependent but should correspond to the largest value which can be stored in the smallest unit of storage for the computer targetted by the implementation.

3-3

The size of a Byte is one Byte.

Words

4-1

Words correspond to the unit of data which the computer directly operates on.

4-2

The maximum possible value which can be stored in a Word is implementation dependent but should be greater than the maximum of a Byte and at least large enough to count the maximum possible size of a unit in the environment.

4-3

The size of a Word is at least two Bytes up to the minimum number of Bytes required to store the maximum Word value.

Ranges

5-1

A Range is an arbitrary sized unit which can store one of a sequence of whole, positive numerical values where the minimum and maximum numerical values are arbitrary.

Places

6-1

Places are derived from Bytes such that for each Byte provided by the computer, there is a unique Place.

Figure 2 6-2

A Place may be derived from any unit in which case the Place corresponds to first Byte in the vector of Bytes occupied by the unit.

6-3

The size of a Place is at least the number of bytes required to store the number counting the bytes available in the environment.

6-4

The default expression which derives a Place from a unit will respect the "provenence" of a unit such that deriving the Place of a unit resulting from a member or field access operation will result in the Place of that member or field.

6-5

A Place may have a duration beyond the unit which it was derived from. In that case the Place will not prevent the unit from being discarded.

6-6

A Place cannot preserve information about the data which begins at that Byte. When accessing the data at a Place, the type of data resulting from the access must be the type which constrains the result. For example, given (x' = (@ "string"), x : Number), the result of (x = (@ x')) must be an instance of Number according to the constraint acting on x.

Vectors

Figure 3 7-1

The vector type is a subtype of the collection type. As such, the terminology used to describe collections is used to describe vectors.

7-2

A vector is a composite type of data where all members are indexed, and instances of the same type.

7-3

Members in a vector are indexed beginning with one for the first member, incrementing by one for each additional member, and the cardinality of the vector being the index of the last member.

7-4

For each unit, the environment will maintain a vector of bytes which contains the data for that unit where the cardinality of the vector is equal to the size of the unit and the Place of the first byte in the vector is the Place of the unit.

Recursion

8-1

Recursion may occur in data where a component of some data may refer to that data. In this case, recursion is only observed by the operations which access components of that data. There will only ever be a single data unit. Executing operations which access data from a component which contains a recursive reference will only yield the existing data unit which contains the recursive component.

Sizes

Figure 5 9-1

The default expression for the size of a unit will return the size as a Word value.

Equivalence

10-1

Equivalence between units is a primitive notion and depends on the default type of the unit.
