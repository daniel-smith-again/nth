Data

Data is the medium with which effects in the environment are recorded. Thus, evaluating a program will produce effects in the environment in the form of a data result.

The forms of data available in nth are...

  • Boolean
  • Byte
  • Word
  • Character
  • Place
  • Array
  • Range
  • Datatype
  • Symbol

Booleans

The Boolean is a form of data that corresponds to a value in binary logic.

The symbols, true and false correspond to boolean 1 and 0 respectively.

All values which are true are equivalent.

All values which are false are equivalent.

Bytes

The Byte corresponds to the smallest unit of storage which can be set in the environment.

A Byte will contain a numerical value of a range such that a Byte corresponds to (from 0 to η).

The maximum possible value, η, which a Byte may contain corresponds to the enumeration of the number of ways in which information can be arranged within a single Byte in the environment. For example, in an environment where the smallest unit of storage is eight bits, Byte would be defined as (from 0 to (2 ^ 8)).

Any two Bytes which contain identical numerical values are equivalent.

Byte Constructor

The Byte Constructor is a default function which accepts a single range parameter.

The result of applying the Byte Constructor to a range will be a Byte value equal to nmodη where n is the range parameter.

Words

The Word corresponds to the basic unit of information with which operations are executed.

A Word will contain a numerical value of a range such that a Word correpsonds to (from 0 to φ).

The maximum possible value, φ, which a Word may contain corresponds to the enumeration of the number of ways in which information can be arranged within a single Word in the environment. It must be greater than the maximum value of a Byte.

Any two Words which contain identical numerical values are equivalent.

Word Constructor

The Word Constructor is a default function which accepts a single range parameter.

The result of applying the Word Constructor to a range will be a Word value equal to nmodφ where n is the range parameter.

Characters

The Character is a range which specifically corresponds to a text encoding. The text encoding correspondence is implementation dependent.

A Character will contain a numerical value of a range such that a Character corresponds to (from 0 to ψ).

The maximum possible value, ψ, which a Character may contain corresponds to the number of Unicode® code points.

Any two Characters which contain identical code points are equivalent.

Character Constructor

The Character Constructor is a default function which accepts a single parameter which is either a range or a string.

The result of applying the Character Constructor will be a Character value equal to nmodψ where n is the range parameter, or it will be the code point corresponding to the Character at the first index in the string parameter.

Places

The Place corresponds to a unique value representing the location of a unit in the environment.

A place does not store a numerical value and does not correspond to a range.

A place must be able to represent a number of unique values up to the number of individual bytes which can be set in the environment. Each of these values must be distinct.

A Place value can only be equivalent to itself.

Referencing Data

Given an environment, there is only ever a finite number of possible Place values. Therefore a place is not constructed, but derived from an instantiated unit. The default function Reference, bound to the identifier @ will accept a single parameter of any type. Applying Reference to anything will result in the Place of the first Byte of the section of storage claimed by that parameter.

Dereferencing Data

A Place only corresponds to a single Byte in storage which carries no information about the actual unit residing at that byte in the environment. The default function Dereference, bound to the identifier @, will accept a single parameter which must be a Place. Applying Dereference to a Place will result in the unit which satisfies the constraint on the result of the Dereference application.

It is considered an error if the size of the Dereferenced unit cannot be derived from the constraint on the result of the Dereference application.

Arrays

The Array is a data form which will store some number of units which must all be instances of a single type. A unit which is a component of an Array is referred to as an element. The type which all elements must be an instance of is referred to as the order. The number of elements contained in the Array is referred to as the cardinality.

Because an Array contains fields, it is a subtype of Collection. However, it is a distinct form of data and does not require support for all the operations which are allowed on Collections.

Any two Arrays are equivalent where the order and cardinality are equivalent, and the values of like indicies are each equivalent.

Array Constructor

The Array Constructor is a default function which takes one or two parameters which are a type or a type and a range respectively. Applying an Array Constructor to a type will result in an Array with an order equal to the parameter, which is the supertype of any Array with an equivalent order and a definite cardinality.

The collection operation, Subscript will also support Arrays. The result of a subscript on an Array with an indefinite cardinality will be the order of that Array.

Numbers

A Number is a form of data which contains a numerical value within an arbitrary fixed range over the integers.

The value stored by a Number cannot exceed the minimum or maximum for that number. Given a value to be stored by a number which exceeds a boundary, the number will "saturate". In other words, the value stored by the number will be the minimum or maximum value, whichever is closest to the excessive value. This is such that ((100000 : (from -10 to 10)) = 10).

The default value for a number must be the value with the least magnitude relative to the integers. In other words, the value which is closest to zero. For example, the default value of (from 8 to 37) is 8 and the default value of (from -394 to 87362) is 0. This is such that the value of n in (x : (from -394 to 87362)) is 0.

Any two Numbers are equivalent if the values they each hold are equivalent relative to the integers. For example (5 = 5) and ((27 : (from -378 to 900)) = (27 : (from 15 to 28))) both result in true

Number Constructor

The Number Constructor is a default expression which will construct a number using two numbers as the minimum and maximum bounds. The first number must always be less than the second number relative to the integers.

Bounds

The default function Bounds will accept a single parameter which must be a number. Applying Bounds to a number will result in a collection where the first field is the number corresponding to the minimum value for the number parameter and the second field is the number corresponding to the maximum value for the number parameter.

The collection resulting from applying Bounds will bind the first field to min and the second field to max.

Symbols

Symbols are a form of data derived from a source which represent unique abstract values.

A symbol is constructed by quoting a program which consists of only a symbol. For example, 'abracadabra constructs a symbol.

A Symbol is only equivalent to itself.

Symbol

The default function, symbol accepts a single parameter which is either a symbol or a string. Applying this function to a symbol will result in the string corresponding to the source from which the symbol was derived. Applying this function to a string will result in the symbol which would be derived from that string if it were a source.

Data Operations

Size

Size is a default function, bound to the identifier ##, which accepts a single parameter of any type. Applying Size to a unit will result in a word value corresponding to the number of bytes required to store the parameter in the environment.

Comparison

Comparison is a default expression which will observe the equivalence of two units and will result in a boolean value confirming or denying equivalence.

Two units cannot be equivalent if they are not instances of the same type.
