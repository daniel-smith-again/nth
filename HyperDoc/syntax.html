Syntax

Minimal Required Character Set

1-1

A conforming implementation must support the Unicode® character encoding as described in 3.2 Conformance Requirements of the Unicode® Standard.

1-2

A conforming implementation will not implement or accept multidirectional text as described in 3.2 Conformance Requirements, Bidirectional Text of the Unicode® Standard. Source text must be accepted as left-to-right order.

1-3

A conforming implementation must not implement nor apply transformations to the source text according to 3.2 Conformance Requirements, Unicode Algorithms and 3.2 Conformance Requirements, Default Casing Algorithms of the Unicode® Standard.

1-4

If an otherwise conforming implementation must function on a system which does not allow support for the Unicode® Standard, the ASCII Standard may be used instead.

Grammar

The Program Model

3-1

An nth program is represented by a Program Model. The Syntax grammar described in Fig. 1 is used to form a written, literal representation of a Program Model, a source.

3-2

A subset of grammar names in Fig. 1 correspond to Data or Program Model structural elements, these are referred to as syntax.

The names which correspond to Data are...

  • Symbol
  • Number
  • String
  • Quote

The names which correspond to Program Model structure are...

  • Expresion
  • Collection
  • Type

The correspondence between source and Program Model is such that the terms, Symbol, Number, String, Quote, Expression, Collection, and Type, as they appear in this document, may interchangeably refer to their appearance as data or syntax in a source.

3-3

Symbols, Numbers, Strings, and Quotes imply a constant data instance at the position where they occur in a source text.

3-4

Expressions, Collections, and Types imply a Program Model structure at the position where they occur.

Constructing Expressions

4-1

Expressions are collections with members which are either data or further nested Program Models.

4-2

Each member of an expression resides at the index corresponding to the order in which it occurred in the source.

4-3

The closed pair of round braces in the expression syntax in Fig. 1 serve as identifying punctuation for expressions. Thus an expression will always begin with an opening round brace and end with a closing round brace.

Constructing Collections

5-1

Collections are an invocation of the default function Elements.

5-2

Each occurrance of syntax within the collection must be included as an element of that collection.

5-3

The closed pair of curly braces in the collection syntax in Fig. 1 serve as identifying punctuation for collections. Thus a collection will always begin with an opening curly brace and end with a closing round brace.

5-4

The collection syntax must be immediately rewritten, wherever it occurs, to an expression which applies the default function Elements using the elements as parameters. This step must preserve the order in which the elements occurred in the collection syntax.
