Evaluation

Evaluation determines the result of a program through a process of repeated intermediary steps which approach a final data result.

Given an expression, there are two steps which are performed to determine the result, Rewriting and Reduction.

Rewriting

An expression may either correspond to a default expression, or an expression which can be transformed.

An expression can be transformed if it is a subtype of the function which binds that expression as a value.

If an expression is not a default expression, then the rewrite step will be performed.

If a rewrite step is performed, it will check the image for a transform which can be applied to the expression.

If no transforms can be applied to the expression, it is considered an error.

If an expression with one or more nested expressions is rewritten, each inner expression must be rewritten before the outer expression.

Reduction

A program can be reduced only when all expressions in the program are default expressions.

Reduction begins by attempting to execute the top, outermost expression in the program.

Reduction of a program is performed according to the definition of the default expressions and default functions.

Contexts

The Context is used to persist data for one or more operations.

Contexts may be arbitrarily nested where the the inner context closes over the outer context.

If a new context is created, it must always close over the active context.

A context may be "active" in which case the bindings it contains are available to the process of evaluation.

Only one context may be active at a time. Executing some default functions may change which context is active.

When a context is discarded, the bindings and the data forming identifiers and values are also discarded.

If a binding is present in a context through a closure, it cannot be discarded when the closing context is discarded.

Control

There are a number of default expressions which do not correspond to operations, but instead affect how a program is evaluated, in other words, they control the "flow" of evaluation. They are...

  • sequential evaluation
  • modulated evaluation
  • conditional evaluation
