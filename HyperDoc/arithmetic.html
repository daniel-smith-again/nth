Arithmetic

Arithmetic is a class of operations, corresponding to mathematical arithmetic, which are generalized across multiple forms of data which contain numerical values.

The arithmetic operations supported by nth are...

  • addition
  • subtraction
  • multiplication
  • division
  • remainder
  • modulus
  • absolute value
  • exponentiation
  • root
  • Conjunction
  • Disjunction
  • Negation

Arithmetic operations will produce different results depending on the data involved in the operation.

Arithmetic over numbers will always result in a number with the least minimum and greatest maximum boundaries. Given (let x : (from 0 to 2) y : (from -4 to -1) in (x = 1) + (y = -3)), the result will be (-2 : (from -4 to 2)).

Arithmetic over Words and Byte is modular by the maximum value respectively. For example, if the maximum value of a byte is 20 then (let x = (byte 15) y = (byte 15) in x + y) results in 10.

Numerical Arithmetic

Addition

Addition is a default expression that will perform numerical addition on the values contained in either a Byte, Word, or Number.

The parameters to a addition must be instances of the same type.

It is considered an error if the two parameters to addition are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Subtraction

Subtraction is a default expression which will perform numerical subtraction on the values contained in either a Byte, Word, or Number.

The parameters to subtraction must be instances of the same type.

It is considered an error if the two parameters to subtraction are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Multiplication

Multiplication is a default expression which will perform numerical multiplication on the values contained in either a Byte, Word, or Number.

The parameters to multiplication must be instances of the same type.

It is considered an error if the two parameters to multiplication are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Division

Division is a default expression which will perform numerical division on the values contained in either a Byte, Word, or Number.

The result of division is a collection containing the quotient and the remainder which must be an instance of (collection quotient = (type Byte Word Number) , remainder = (type Byte Word Number)).

If the constraint on the result of division is a subtype of (type Byte Word Number), then the first field of the resulting collection must be used as the result of the division. This is such that the result of (1 + (7 / 3)) is 3 but the result of ((collection _) + (7 / 3)) is (collection quotient = 2 , remainder = 1)

It is considered an error if the two parameters to division are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Remainder

Remainder is a default expression which accepts a result from division and will extract the field containing the remainder value.

Modulus

Modulus is a default expression which will perform numeric modulo on the values contained in either a Byte, Word, or Number.

The parameters to modulus must be instances of the same type.

It is considered an error if the two parameters to modulo are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Absolute Value

Absolute value is a default expression which will result in the absolute value of any number given as a parameter.

Exponentiation

Exponentiation is a default expression which will perform numerical exponentiation on the values contained in either a Byte, Word, or Number.

The parameters to exponentiation must be instances of the same type.

The first value will be the base and the second value will be the power.

It is considered an error if the two parameters to subtraction are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Root

Root is a default expression which will perform numerical nth root on the values contained in either a Byte, Word, or Number.

The parameters to root must be instances of the same type.

The first value is the index and the second value is the radicand.

It is considered an error if the two parameters to root are not instances of the same type except when two numbers are provided as parameters in which case they must only be instances of Number.

Boolean Arithmetic

Boolean Arithmetic is a class of three default expressions which correspond to boolean arithmetic in classical logic.

Conjunction

Conjunction will perform logical conjunction on two boolean values.

Disjunction

Disjunction will perform logical disjunction on two boolean values.

Negation

Negation will perform logical negation on a boolean value.

Combined Arithmetic

Combined Arithmetic is a default expression which will rewrite an arithmetic expression formed using the arithmetic expressions above according to the order of operation in classic mathematics.

Combined Arithmetic accepts any expression where all fields are arithmetic keywords or one of Bytes, Words, or Numbers.

Combined Arithmetic will rewrite expressions such that arithmetic is performed according to the following order of operations...

  1. absolute value
  2. exponentiation and root
  3. multiplication, division, modulus, and remainder
  4. addition and subtraction

Combined Boolean Arithmetic

Combined Boolean Arithmetic is a default expression which will rewrite a boolean expression formed using boolean values and the keywords for the Boolean Arithmetic.

Combined Boolean Arithmetic will rewrite expressions such that the boolean operations are performed in order of occurrence.

Comparison

Any two numerical values stored in either a Byte, Word, or Number may be compared.

The comparison operations are as follows...

  • inequality
  • greater than
  • greater or equal
  • less than
  • less or equal

Inequality

Inequality will result in true if the parameters are not equivalent and false otherwise.

Greater

Greater will result in true if the first parameter is greater than the second parameter and false otherwise.

Greater or Equal

Greater or Equal will result in true if the first parameter is greater than or equivalent to the second parameter and false otherwise.

Less

Less will result in true if the first parameter is less than the second parameter and false otherwise.

Less or Equal

Less or Equal will result in true if the first parameter is less than or equivalent to the second parameter and false otherwise.
