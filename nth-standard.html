Introduction

This document contains the specification of "nth", a system for creating and performing computations. This system is intended to be generally useful when applied to any problem which requires computing.

The system is comprised of a description language for composing programs and a small environment to compute those programs. In this document, the name "nth" refers to the description language. The term "environment" shall refer to the mechanisms and resources through which computation is facilitated.

As this document is a specification, all properties and behaviors of the system as described are considered requirements for an implementation.

Syntax

Character Set

nth is written as a sequence of characters. The minimal set of characters required to write an nth program is supported by the printable subset of the ASCII character encoding (all characters excluding control characters). nth is intended to be compatible with any character encoding which provides the same set of printable characters as the ASCII encoding. For the purposes of this standard, the ASCII subset is assumed, with the addition of a line break character.

nth assumes a classification of characters which is applied to any encoding. A character may either be a Digit, Glyph, or Blank. A Digit is a character corresponding to one of the ten decimal digits. A Glyph is any character which is visible when printed. A Blank is any character which results in some blank space or formatting structure when printed.

Grammatical Components

The grammatical components of an nth program are classified as Symbols, Numbers, and Punctuation. These are combined to form a Program Model.

A Symbol is any sequence of characters not including Punctuation or a Blank.

A Number is any symbol which only contains decimal digits. Numbers may optionally include either a fractional or decimal component as a forward slash or period dot respectively. These components cannot occur as the first or last character in the Number. Numbers may optionally include a sign component as a plus or dash. The sign can only occur as the first character in the Number.

Punctuation is any one of the following characters, (){}[],'". A Punctuation character represents grammatical structure and cannot be treated as a Symbol.

Programs

An Expression is a closed pair of round braces surrounding zero or more programs. A Sequence is a series of programs separated by a comma. A Program is a composition of Expressions, Sequences, and Constructors.

The Program Model

Format

A Program Model is some structured data corresponding to a written nth program. It unambiguously represents the data used by a program and the exact order of the manipulations performed on that data.

A Program Model is composed of nested Collections containing sequences of Symbols, Constructors, and other Collections. A Node refers to a Collection used as a structural component of a Program Model. Programs are discretely classified as Operations and Constructors. An Operation is a Program which represents a manipulation performed on some Data. A Constructor is a Program which represents a specific formation of Data.

Construction

There is an exact correspondence between a Program and a Program Model. Expressions, Sequences, Collections, and Types all correspond to individual Nodes within a Program Model. If an Expression, Sequence, Collection, or Type is nested within another in a Program, then the resulting Nodes in the Program Model will be nested. Computation

Computation, in terms of the nth langauge, is the process of transforming a Program Model to a data result through repeated reductive steps. A single reductive step will determine the first Node in a Program Model which does not contain nested Nodes and reduce a Node to some data. Environment

Overview

The nth environment consists of some finite number of Data units which can store values.

Each unit can be accessed through a Reference which uniquely identifies that Data unit.

Constructors and Operations will claim resources from the environment in the form of some number of Data units.

Data

Data is a type corresponding to the smallest individual unit of storage provided by the environment. A Data unit can store one of a number of abstract values. The number of values which can be stored as Data are dependent on the nature of the storage medium supplied by the environment. nth enumerates these values as a set of sequential numerical values beginning with zero and ending with a number which is one less than the cardinality of the set.

There is a unique Reference value for each unique Data unit available in the environment.

Operations on Data

Data is constructed by converting a number corresponding to a Data value to a format which can be stored as Data. Values can also be computed using a modular arithmetic over existing Data.

Logic

Collections

A Collection is a structured entity which represents a grouping of other entities.

Types

The nth langauge provides a system of Types which allows for constraints to be placed on what values may occurr at any point in a program. A Type is a potentially infinitely large Collection of unique entities enumerating all possible entities which may occur where that Type occurs. The entities grouped as a Type are referred to as values. Types may be values of other Types.

A Type is constructed by listing any number of programs in between the two punctuation characters [ and ]. The form of that type is then the values that result from computing each of the programs.

Typed Programs

A program can only be computed if it is considered well-typed. A well-typed program is a program which contains no type errors.

A program will have no type errors if it conforms to a set of Type rules. These rules are given in the figures below.

Quantification

Quantification is a process which computes a program once for each value of a Type and constructs a new Type representing the values resulting from the quantification. Quantification is engaged by creating an independent variable. An independent variable is a variable which has been constrained by a type, but is not bound to a single value. Any program featuring an independent variable is quantified.
