Syntax

nth syntax is formed using expressions, symbols, and strings. Expressions denote "doing", while symbols and strings denote "being". Expressions may be nested, enabling the "doing" of multiple things within a single program. An expression which isn't nested may be called a program, since it usually encapsulates a set of behaviors. A nested expression may be referred to as a function or sub-program.

nth syntax is utf-8 encoded and can be written using any valid Unicode® character. Some categories of characters are discarded in order to avoid obfuscating a program and potentially causing unexpected behaivor. Generally, all visible (printable) Unicode® are accepted in nth syntax.

Symbol Syntax

Symbols are comprised sequences of any character which isn't a round brace, double quote, or blank space.

String Syntax

Strings are comprised of double quotes surrounding a sequence of arbitrary characters. A string will ignore any character which is a formatting character, such as line breaks or horizontal tabs.

A string may optionally contain an "escape sequence" which is an additional syntax relevant only between a pair of double quotes. All escape sequences begin with a backward slash. There are three escape sequences required by nth, a backslash escape, a quotation escape, and a byte escape.

The backslash escape is a backslash followed by a second backslash, \\ The initial backslash is discarded and the second backslash is included as a character in the string.

The quotation escape is a backslash followed by a double quote, \". The initial backslash is discarded and the quotation is included as a character in the string.

The byte escape sequence is a backslash followed by exactly two hexadecimal digits. The initial backslash is discarded and the two hexadecimal digits are interpreted as a byte value to include at the position where a character would be included in the string. The hexadecimal digits accepted in a byte escape sequence are 0123456789abcdefABCDEF.

Expression Syntax

Expressions are written using round braces which surround a series of symbols, strings, and other expressions. Two adjacent symbols in an expression must always be separated by one or more blank space characters.

Character Escapes

The Unicode character set is expansive and not all graphemes are easily accessible from a normal keyboard. To allow greater unicode accessibility, some characters are expressible using a character escape sequence, which is an ASCII character sequence corresponding to a Unicode® grapheme.

A character escape is a symbol which begins with a single backslash followed by a name written in the ASCII alphabetical characters. That symbol will be replaced with the corresponding Unicode® grapheme.

Grammar

Parsing Rules

A program may be represented not only in a written form with syntax, but as structured data which may be manipulated by another program. Therefore nth syntax is required to have a strict correspondence to structured program data.

The nth syntax grammar is categorized into three distinct correspondences, symbols, data constants, and expressions.

A symbol corresponds to any contiguous sequence of characters formed according to the Symbol production rule in the grammar. A symbol will include all characters up until the first Whitespace, Quotation, or Begin production rule occurs.

An expression corresponds to any contiguous sequence of characters formed according to the Expression production rule. After the Begin sequence occurs, each individual sequence occurring before the End sequence will be included as an atom in the expression. The order of the atoms in an expression must exactly match the order in which their corresponding sequences occurred.

Sessions

Programs in nth are created through the process of "image-based development". Rather than a program being represented by an ad-hoc collection of source text files, Image-based development involves a reactive system which maintains all the program definitions and data constants in a live process.

A session is minimally a duplex stream whereby the programmer submits programs to the system and the result are sent back to the programmer as if passed as an argument to print. Sessions may provide alternative interfaces with extended capabilities or interactivity, as long as they conform to the notion of sending and recieving nth programs.

Modules

Modules are the construct which allows program and data created in a session to be stored, recalled, and otherwise preserved in a live system. They are essentially lists of bindings. A binding is an association of a symbol to some data, such that when that symbol occurs in a program, it can be replaced with that data.

Default Modules

Within a live session, there are a number of implicit modules which are used to facilitate the capabilities of the nth language.

Default

The default module is an implicit module which contains all nth language definitions. All primitive macros, functions, and constants are members of the default module.

The default module is immutable and cannot be modified by the programmer.

The default module must always be present at the start of a session and cannot be discarded.

Top Level

The default module contains a nested, unnamed module referred to as the Default Top Level. This is the context in which all code submitted to the session is evaluated. It serves to guard the default module from modification.

Leaving the context of the Top Level will exit the session.

If another module is entered using in, that module will be treated as the Top Level although it is not the Default Top Level. All programs which modify the Top Level will modify that module.

Primitive Macros for Modules

There are seven primitive macros which affect the Top Level when evaluated. They are define, use, syntax, data, in, export, and leave.

Definition

The Definition primitive macro is used to modify the list of bindings held in a module. It is invoked with the symbol, define, followed by a symbol, followed by a sub-program. The symbol is used as the binding variable. The sub-program is evaluated to yield the data which will be the subject of the binding.

Use

The Use primitive macro is used to introduce bindings from other modules into the context of the Top Level. It is invoked with the symbol, Use, followed by one or more import expressions.

Use will transfer bindings from a module to the Top Level context. When a binding is transferred, the same variable is used and will bind the instance of data which is held in the module rather than copying it.

Use may optionally transfer a binding using a nickname which is a symbol which will be used in place of the variable in the transferred binding in the Top Level.

An import expression is one of

  • A string corresponding to a filessytem path where the module is stored.
  • A symbol corresponding to the name of a module file stored in one of the default paths.
  • A non-empty expression where each item is an import requirement and the last item is a string or symbol corresponding to a module file.

An import requirement is either a symbol, or an expression containing a pair of symbols. If the requirement is a single symbol, that symbol must be a variable in the module being imported. That symbol will be used as the variable If the requirement is an expression, then the first symbol is a "nickname" which will be used as the variable when the required binding is established in the Top Level. The second symbol must correspond to a variable in the module being imported.

Terminology

Host
The arbiter of computing resources in the environment where an implementation executes programs.
Processor
The agent responsible for executing a program.
Execution
The process by which the meaning of a program is carried out.
Program
The data to be interpreted during execution.
Context
The enumeration of all bindings given a position in a program.
Binding
An association between a symbol and some data such that the data may replace the symbol in a program anywhere that association is present.
Symbol
An opaque data type which is a component of a program.
Expression
A composite data type which is a component of a program.
Function
A program which preserves a context to be used during execution.
Syntax
The sequence(s) of characters which correspond to a program.
Character
An opaque value corresponding to a visually rendered grapheme.
Number
A number.
Evaluation
The reductive process of deriving the next component of a program which execution will interpret.
Session
A channel of communication between the live image and the programmer.
Image
A stored collection of programs and data.
System
A group of related programs which can be extracted from an image to form a stand-alone software application.
Variable
The symbol in a binding which can be replaced with data.
Subject
The data in a binding which will replace a symbol.
Type
An abstract list of members which are either data or other types.
Value
A type which has only a single member.
Subtype
The relation which establishes that an entity is a member of a type. That is to say, a member of a type must be a subtype of that type.
