Overview

This document provides the definition the programming language, "nth", stylized as nth. Specifically, the meaning, interpretation, and formation of programs written in the nth programming language. Information is provided in such a way that the reader is able to rely on this document not only to understand how nth is used, but also how one might implement nth.

Terminology

Host
The arbiter of computing resources in the environment where an implementation executes programs.
Processor
The agent responsible for executing a program.
Execution
The process by which the meaning of a program is carried out.
Program
The data to be interpreted during execution.
Context
The enumeration of all bindings given a position in a program.
Binding
An association between a symbol and some data such that the data may replace the symbol in a program anywhere that association is present.
Symbol
An opaque data type which is a component of a program.
Expression
A composite data type which is a component of a program.
Function
A program which preserves a context to be used during execution.
Syntax
The sequence(s) of characters which correspond to a program.
Character
An opaque value corresponding to a visually rendered grapheme.
Number
A number.
Evaluation
The reductive process of deriving the next component of a program which execution will interpret.

Syntax

The syntax of nth corresponds to the formation and composition of symbols and expressions. In addition there is syntax for describing various forms of data.

Expressions

The syntax for expressions is an opening round brace (, followed by a closing round brace ), forming delimiters around the contents of the expression. An expression is minimally just the two brace characters adjacent to each other.

Symbols

Symbols are comprised of any character which is not a blank space or expression delimiter.

Two symbols which are adjacent are required to be separated using one or more blank spaces, an expression, or a combination of both.

Strings

String data is written by surrounding arbitrary character sequences with a pair of double quotations, ". All visible characters are included as items in the string data, including blank spaces. Formatting characters such as tabs, line breaks, and other control characters are ignored.

String syntax includes an "escape" sequence where values can be written according to a character sequence rather than a character value taken literally. An escape sequence begins with a backward slash, \ followed by a double quotation, a backward slash, or a two-digit hexadecimal sequence.

Hexadecimal digits may only be one of the characters appearing in the following sequence,
0123456789abcdefABCDEF
namely, the ten decimal digits, and A through F in upper and lower-case form.

String Formalization

String data will always correspond to an array of UTF-8 encoded bytes. The data will exist as if constructed with (array Byte)

An escape sequence which gives a hexadecimal value will set the value of a single byte at the index in the array where the escape sequence occurred.

String data will specifically discard any character values included in the Cc character category as specified in the Unicode encoding. Escape sequences are permitted to specify byte values which may be identical to an otherwise discarded character value.

Alternative Expression Syntax

Expressions are cannonically denoted with round brace delimination, however expressions may alternatively be denoted by line breaks and indentation according to the following associations...

  1. A single line corresponds to a single expression.
  2. A line break followed by an identical tab sequence to the previous line will begin a new expression.
  3. A line break followed by a tab sequence longer than the previous line will begin a new expression nested in the expression begun on the previous line.
  4. A line break followed by a tab sequence shorter than the previous line will close the expression on the previous line and continue in the next outer expression.
  5. Any two lines beginning with identical tab sequences correspond to the contents of the same expression.
  6. Two consecutive line breaks correspond to the minimum number of closing round braces required to close the current expression.
  7. Round brace delimiters must be paired only with other round brace delimiters and cannot be paired with indentation and line breaks.

Generally, an nth program may be written in any visual format using line breaks and blank spaces for non-significant indentation. Introducing tab characters directly after a line break will engage the alternative expression syntax.

Grammar

Cannonical Syntax

It's possible to write an nth program without using parentheses. Programs written in this fashion are required to be converted to the cannonical syntax which uses exclusively parentheses to denote expressions. Conversion will happen according to the following rules...

  • A line corresponds to an expression.
  • Any line following a closed expression will begin a new expression and will be prefixed with (.
  • A line break followed by one or more spaces will continue the expression from the previous line.
  • A line break followed by the same number of tabs as the indentation on the previous line will be converted into the sequence )(.
  • A line break followed by less tabs than the previous line will be converted into the sequence ).
  • A line break followed by more tabs than the previous line will be converted into the sequence (.
  • Two consecutive line breaks will be converted into as many closing parentheses, ), as required to close the outermost expression.
  • Any line following a closed expression, but still being indented, will be prefixed with as many opening round braces, (, as required to guarantee the contents of the least indented line are mapped to atoms in the outermost expression.

Parsing Rules

A program may be represented not only in a written form with syntax, but as structured data which may be manipulated by another program. Therefore nth syntax is required to have a strict correspondence to structured program data.

The nth syntax grammar is categorized into three distinct correspondences, symbols, data constants, and expressions.

A symbol corresponds to any contiguous sequence of characters formed according to the Symbol production rule in the grammar. A symbol will include all characters up until the first Whitespace, Quotation, or Begin production rule occurs.

An expression corresponds to any contiguous sequence of characters formed according to the Expression production rule. After the Begin sequence occurs, each individual sequence occurring before the End sequence will be included as an atom in the expression. The order of the atoms in an expression must exactly match the order in which their corresponding sequences occurred.
