for
Nth

0
| This document specifies a system for describing and computing programs called "Nth".

1
| This document represents a standard. The descriptions therein represent requirements for an implementation of Nth.

1.1
| An implementation is said to be "conforming" if it behaves according to all requirements given in this standard.

There are two discrete components to the Nth system, the language and the computational environment. In this document, "Nth" refers specifically to the language component and the environment is referred to by "environment".

Nth is a homoiconic language. The written form of the language coresponds exactly to the literal structure representing the program.

The term "Program Model" refers to the cannonical representation of a program in Nth.

A written program is then homoiconic with the Program Model.

A written program is formed as a sequence of character values.

Nth minimally requires the characters provided by the ASCII character encoding scheme. Other encoding schemes which include the characters provided in the ASCII character set may be used.

Characters are classified into one of three generic groups, Glyphs, Digits, and Blanks.

A Glyph is any character which is expressed visually.

A Digit is a Glyph, specifically one of the ten decimal digits, 0123456789

A Blank is any character which is expressed by empty space between, or the positioning of, some characters.

The following grammar describes all valid sequences of characters forming a program. The symbol, →, denotes a relation between a named position on the left, and a list of elements that can occur at that position given on the right. The symbol, |, separates elements in a list. The symbol, *, denotes an element which may repeat one or more times where it occurs. Characters given in monospace represent themselves literally.
