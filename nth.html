Overview

The nth system is comprised of two discrete components, a language to express programs, and an environment which facilitates computation. The name nth specifically refers to the language component and "environment" will refer to the environment.

Syntax

A program can be represented in two forms, "written" or text form, and the program model. Both forms are equivalent, however the written form of a program includes additional notation for constructing data as well as program structure.

A written program is formed as a sequence of character values.

Characters are grouped into tokens which represent individual syntactic components corresponding to constructions of data or program model structure.

Expressions, Sequences, Collections, and Types correspond to program model structure. Strings, Quotes, Symbols, Data, and Numbers correspond to constructors.

Expressions, Sequences, Collections, and Types are all represented by a collection containing data or nested programs as elements.

The String notation constructs a vector of characters exactly as long as the number of characters occurring between a pair of double quotes.

The backslash is used to explicitly include characters in a string. If the backslash is followed by another backslash or a double quote, a backslash or double quote is included in the string respectively. If the backslash is followed by a sequence of numbers, those numbers are interpreted as the numeric value corresponding to a value in the character encoding scheme used by the implementation.

The backslash prefixes some expression representing a character which will be included in the string at the position where the backslash occurs.

If a backslash is followed by another backslash, then a backslash is included in the string.

If a backslash is followed by a double quote, then a double quote is included in the string.

If a backslash is followed by an expression, then the expression is computed

The Program Model

The structured data which represents a program is referred to as a "program model".

The program model unambiguously represents all aspects of a program.

A program model is comprised of collections representing individual computational steps. The elements of each collection are used to determine how the expression is computed.

Computation

Environment

The environment manages the resources required for computation

Resources in the most general sense means storage space for holding data.

The grammar of nth provides a notation for describing the stored format of data block.

Units, Words, and References all store a value which is a natural number inclusively between zero and the maximum magnitude allowable by the size of each type.

Numbers

A Number is an entity which represents a quantity corresponding to a value in the set of rational numbers.

nth provides a grammar for constructing numerical values. A number is expressed as a single sequence of digits, or two sequences of digits with a decimal or fractional separator.

A number expressed as a single sequence of digits corresponds to the base-ten decimal notation for a whole positive number.

A number expressed as two sequences separated by a decimal separator (.) corresponds to the base-ten decimal fraction notation for a rational positive number.

A number expressed as two sequences separated by a fractional separator (/) corresponds to the rational notation of a fraction between two whole positive numbers.

The decimal and fractional notations of numbers are equivalent up to infinite decimal expansion.

A rational number with an infinite decimal expansion may only be expressed using fractional notation.

addition will compute to the sum of the two numbers given in the expression.

sum will compute the sum of all numbers given in the expression.

subtraction will compute the difference taking the left number as the minuend and the right number as the subtrahend.

sequential subtraction will compute the difference taking the left-most pair of numbers with the very left as the minuend and the second-left as the subtrahend, and replacing them with their difference until no more pairs are possible.

multiplication will compute the product of the two numbers given in the expression.

sequential multiplication will compute the product taking the left-most pair of numbers and replacing them with their multilicative product until no more pairs are possible.

division yielding quotient will compute the quotient taking the left number as the dividend and the right number as the divisor.

sequential division will compute the quotient taking the left-most pair of numbers with the very left as the dividend and the second-left as the divisor, and replacing them with their quotient until no more pairs are possible.

division yielding modulus will compute the remainder taking the left number as the dividend and the right number as the divisor.

A type constraint may be placed on a variable, or a program.

0This document specifies a system for describing and computing programs called "Nth".

1This document represents a standard. The descriptions therein represent requirements for an implementation of Nth.

1.1An implementation is said to be "conforming" if it behaves according to all requirements given in this standard.

There are two discrete components to the Nth system, the language and the computational environment.In this document, "Nth" refers specifically to the language component and the environment is referred to by "environment".

A written program is formed as a sequence of character values.

Nth minimally requires the characters provided by the ASCII character encoding scheme. Other encoding schemes which include the characters provided in the ASCII character set may be used.

Characters are classified into one of three generic groups, Glyphs, Digits, and Blanks.

A Glyph is any character which is expressed visually.

A Digit is a Glyph, specifically one of the ten decimal digits, 0123456789

A Blank is any character which is expressed by empty space between, or the positioning of, some characters.

A special case which is not represented in this grammar is that when a symbol or number immediately follows another symbol or number, a Delimiter must occur between the two.

The "program model" is the cannonical representation of a program.

The structure of a program model is identical to the grammatical structure of a written program.

The grammatical forms, Expression, Sequence, Collection, Type, Quote, and Delimiter represent the substructures which form a program model.

The grammmatical forms, String, Symbol, and Number represent constructions of data and are not a structural component of a program model.

An Expression is a collection of entities and other nested expressions representing a computational step.

A Sequence is a collection of entities and nested programs which are computed in sequential order.

A Collection is a collection of entities and nested programs that constructs a collection of entities.

A Type is a collection of entities and nested programs which constructs a type.

A Quote is a nested program which is interpreted as structured data rather than a computable part of the program model.

A Delimiter unambiguously separates two grammmatical elements in a written program, but is not included in a program model.

A String is a form of vectored data containing only characters.

A Symbol is a String interpreted as a single abstract value.

A Number is a Symbol interpreted as a numerical value.

A collection is a structural type which represents an association between one or more separate entities.

Entities which are associated in a collection are referred to as "elements".

Each element in a collection is counted and can be referred to by a unique number.

A collection does not constrain the physical location where an element may be stored in the environment.

A collection forms a fresh context which closes over elements.

A collection is constructed by writing a number of programs in between a closed pair of curly braces.

The programs in a collection constructor expression are computed in the order in which they are written.
