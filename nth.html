0 This document specifies a system for describing and computing programs called "Nth".

1 This document represents a standard. The descriptions therein represent requirements for an implementation of Nth.

1.1 An implementation is said to be "conforming" if it behaves according to all requirements given in this standard.

There are two discrete components to the Nth system, the language and the computational environment. In this document, "Nth" refers specifically to the language component and the environment is referred to by "environment".

Nth is a homoiconic language. The written form of the language coresponds exactly to the literal structure representing the program.

The term "Program Model" refers to the cannonical representation of a program in Nth.

A written program is then homoiconic with the Program Model.

A written program is formed as a sequence of character values. Nth minimally requires the characters provided by the ASCII character encoding scheme.

The character sequence is grouped into discrete tokens. Tokens are either Symbols, Numbers, Blanks, or Punctuation.

A Symbol is a token which does not contain any Blanks or Punctuation.
